Руководство: 

В config_loader указан путь к конфиг-файлу. Сам конфиг в json'е, так удобнее его в питон загрузить было. Все пути берутся из конфига. 
.keymap - это мой кеймап. В Х11 генерируется с помощью xmodmap -pke. xmodmap не выдает сам маску для мыши, поэтому надо в keymap дописать
  строчку из девяци цифр вида "pointer 1 2 3 4 5 6 0 0 0", где вы укажите номера своих кнопок на соответствующих местах(в примере шесть кнопок)
hooking.py - это собственно сам демон. Его надо запускать с sudo правами.
clonner.py - этот процесс надо запустить в пространстве иксов, от него будут форкаться все приложения, которые демон будет запускать.
  Сам демон не может этого делать. Банального запуска в консоле хватит. Но я, напрмер, добавил его в автостарт kde, чтоб его не было видно.
menu.py - это процесс меню, которое будет запускать демон.
Итак, демон ведет лог нажатий клавиш и кнопок мыши. На F1 вызывается меню для последней нажатой клавиши. В меню добавляются действия для текущей
  клавиши. Для симуляции будет выбрана предпоследняя клавиша, т.е. нажав на 'z' потом на 'x' потом на 'c' и на F1 и выбор симуляции, то
  нажимая на 'c' теперь будет эмулироваться еще и нажатие на 'z'. Команды run и kill работают с именами приложений из bin/
  Блокированная клавиша не будет работать в иксах, однако будет писаться в event файл, а значит ее будет ловить демон. Так что функционал,
  который был добавлен к кнопке до блокировки, продолжит работать. Это фича такая.
На F2 демон будет выключен
В конфиге можно указать stdout и stderr демона. Он выводит небольшой лог в stdout. В stderr пишутся ошибки, о них ниже.
